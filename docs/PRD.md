# 小兔子数学乐园 — 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
小兔子数学乐园

### 1.2 产品定位
面向 5-7 岁儿童的趣味数学练习 H5 小游戏，以可爱的小兔子为主题角色，通过游戏化的答题机制让孩子在玩乐中完成每日数学练习。

### 1.3 目标用户
- 主要用户：5-7 岁学龄儿童
- 辅助用户：家长（协助启动游戏、查看成绩）

### 1.4 核心目标
- 将每日 80 道数学练习变成有趣的闯关游戏
- 通过即时反馈和鼓励机制保持孩子的学习积极性
- 适配手机和平板，随时随地可以练习

## 2. 功能需求

### 2.1 难度选择
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-01 | 进入游戏后显示难度选择界面 | P0 |
| F-02 | 提供三种模式："20以内加减法"、"50以内加减法"、"100以内加减法" | P0 |
| F-03 | 界面预留扩展位，方便后续增加"乘除法"等模式 | P1 |

### 2.2 题目生成
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-04 | 题型包含：加法、减法、进位加法、退位减法、连加连减（2个运算符）、长连加连减（3个运算符） | P0 |
| F-05 | 所有中间结果和最终答案必须在 [0, max] 范围内（max 取决于所选模式） | P0 |
| F-06 | 同一关内不出现重复题目 | P0 |
| F-07 | 题目分布：约 20% 加法、20% 减法、15% 进位/退位、27% 连加连减(2-op)、18% 长连加连减(3-op)，连加连减总占比 ≥ 45% | P0 |
| F-08 | 答案分布尽量均匀，避免集中在某个区间 | P1 |
| F-09 | **题目难度下限**：过于简单的题目不出现。加法两个操作数均 ≥ 3，减法被减数 ≥ 5 且减数 ≥ 2，连加连减各操作数 ≥ 2 | P0 |

### 2.3 答题流程
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-09 | 屏幕每次显示一道题目 | P0 |
| F-10 | 通过屏幕数字键盘（0-9）输入答案 | P0 |
| F-11 | 提供删除键和确认键 | P0 |
| F-12 | 答案最多输入的位数根据模式决定（20以内2位，50以内2位，100以内3位） | P0 |
| F-13 | 答对后自动跳到下一题（间隔约 1.5 秒） | P0 |
| F-14 | 答错第一次：提示"再试一次"，倒计时重置，清空输入 | P0 |
| F-15 | 答错第二次：显示正确答案，2 秒后自动跳到下一题 | P0 |
| F-16 | 倒计时耗尽视为答错，进入答错流程 | P0 |

### 2.4 关卡系统
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-17 | 每关 20 题，共 4 关（合计 80 题） | P0 |
| F-18 | 每关结束显示结算画面：正确率、平均用时 | P0 |
| F-19 | 结算画面提供"继续下一关"和"休息一下"两个选项 | P0 |
| F-20 | 选择休息时显示休息画面，点击"准备好了"继续 | P0 |
| F-21 | 4 关全部完成后显示总结算画面 | P0 |
| F-22 | 总结算画面提供"再玩一次"按钮回到开始页 | P0 |

### 2.5 倒计时
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-23 | 每道题限时 15 秒 | P0 |
| F-24 | 倒计时以进度条或圆环形式可视化展示 | P0 |
| F-25 | 剩余 3 秒时进度条变为红色并闪烁提醒 | P1 |

### 2.6 进度展示
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-26 | 游戏画面顶部显示当前关卡和题号（如"第1关 3/20"） | P0 |
| F-27 | 圆形进度环显示当前关卡完成进度 | P0 |

## 3. 视觉设计

### 3.1 整体风格
- 活泼可爱的卡通风格
- 柔和的粉色系/暖色系配色
- 圆角设计，无尖锐元素
- 大号字体和按钮，适合儿童操作

### 3.2 小兔子角色
- 作为游戏伴侣角色，始终在答题画面中出现
- **身体结构一体化**：头部和身体通过脖子区域自然连接，不能出现分离/断开的视觉效果
- 状态表情：
  - **空闲**：轻轻上下浮动
  - **答对**：开心跳跃，眼睛变成 ^_^
  - **答错**：歪头鼓励，略显关切
  - **庆祝**：举起双手，快速蹦跳（关卡完成时）

### 3.3 动画效果
- 答对时：屏幕飘落彩色纸片（confetti）
- 答错时：题目区域轻微抖动
- 切换题目时：淡入上滑动画
- 关卡完成时：星星爆炸效果

### 3.4 配色方案
| 用途 | 颜色 | 色值 |
|------|------|------|
| 主背景 | 淡粉色 | #FFF5F5 |
| 次背景 | 淡桃色 | #FFF0E6 |
| 强调色 | 粉色 | #FF8AAE |
| 正确 | 绿色 | #4CAF50 |
| 错误 | 红色 | #FF6B6B |
| 文字 | 深棕色 | #5D4037 |
| 倒计时安全 | 薄荷绿 | #7DDCBA |
| 倒计时警告 | 黄色 | #FFD93D |
| 倒计时危险 | 红色 | #FF6B6B |

## 4. 音效设计

### 4.1 语音鼓励 (TTS)
使用浏览器 Web Speech Synthesis API 生成中文语音，**优先选择女声**，语调活泼温暖，有感情有鼓励感。话术轮换避免重复：

**答对（一次答对）：**
- "太棒了！"
- "你真厉害！"
- "答对了！好聪明！"
- "非常好！"
- "就是这样！"
- "你是最棒的！"
- "继续加油！"

**答对（第二次答对）：**
- "不放弃，真棒！"
- "坚持就是胜利！"
- "好样的！"

**答错（第一次，鼓励重试）：**
- "再试一次哦！"
- "别着急，再想想！"
- "加油，你可以的！"

**答错（两次都错）：**
- "没关系，下一题加油！"
- "没关系，下次一定行！"
- "记住答案，下次就会了！"

**倒计时耗尽：**
- "时间到啦！"
- "要快一点哦！"

**关卡完成：**
- "太厉害了！一关通过！"
- "你真棒！休息一下吧！"

### 4.2 音效 (Web Audio API)
| 场景 | 音效描述 |
|------|----------|
| 答对 | 上行两音阶（叮咚） |
| 答错 | 低沉下行音（噔噔） |
| 按键 | 短促清脆点击音 |
| 倒计时最后3秒 | 轻柔滴答声 |
| 关卡完成 | 上行琶音（哆咪嗦哆） |
| 全部完成 | 欢快庆祝旋律 |

### 4.3 背景音乐
- 使用 Web Audio API 生成简单循环旋律
- C大调五声音阶，节奏轻快
- 音量极低（作为氛围底衬）
- 提供开关按钮

## 5. 交互流程图

```
开始页面（选择难度）
    │
    ▼
游戏页面（第1关，第1题）
    │
    ├─ 答对 → 庆祝动画 → 1.5s后下一题
    ├─ 答错(第1次) → "再试一次" → 重置倒计时
    ├─ 答错(第2次) → 显示正确答案 → 2s后下一题
    └─ 倒计时耗尽 → 视为答错
    │
    ▼ (20题完成)
关卡结算页面
    │
    ├─ "继续下一关" → 游戏页面（下一关）
    └─ "休息一下" → 休息页面 → "准备好了" → 游戏页面
    │
    ▼ (4关全部完成)
总结算页面
    │
    └─ "再玩一次" → 开始页面
```

## 6. 设备兼容

### 6.1 支持设备
- 手机（iOS Safari、Android Chrome）
- 平板（iPad Safari、Android Chrome）

### 6.2 响应式要求
- 手机端：竖屏全屏布局，按钮不小于 48px
- 平板端：居中显示，内容区域最大宽度 600px
- 禁止缩放（防止误触双击放大）

---

## 7. 技术约束
- 纯前端实现，单个 HTML 文件
- 无外部依赖（不依赖CDN、第三方库）
- 浏览器打开即可使用，无需安装
