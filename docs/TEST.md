# 小兔子数学乐园 — 测试文档

## 1. 测试策略

由于本项目是纯前端单 HTML 文件，测试分为两类：
- **自动化测试**：通过内嵌测试脚本，在浏览器控制台中运行，验证核心逻辑（题目生成、状态管理）
- **手动测试**：验证 UI 交互、音频、动画、响应式布局等

## 2. 自动化测试用例（题目生成算法）

以下测试可通过浏览器控制台运行，或在 HTML 中内嵌 `<script>` 测试块实现。

### TC-01: 加法题生成 — 结果在 [0, max] 范围内
```
对每种模式 (within20/50/100) 各生成 500 道加法题
断言：每道题的 answer ∈ [0, max]
断言：每道题的所有 operands ≥ 3（过滤极简题）
断言：operands[0] + operands[1] === answer
```

### TC-02: 减法题生成 — 结果非负且不过于简单
```
对每种模式各生成 500 道减法题
断言：每道题的 answer ≥ 0 且 ≤ max
断言：operands[0] ≥ 5（被减数不过小）
断言：operands[1] ≥ 2（减数不过小）
断言：operands[0] - operands[1] === answer
```

### TC-02b: 进位加法生成
```
断言：answer ∈ [0, max]
断言：计算正确
```

### TC-02c: 退位减法生成
```
断言：answer ∈ [0, max]
断言：计算正确
```

### TC-03: 连加连减题生成（2运算符）— 中间结果和最终结果合法
```
对每种模式各生成 500 道连加连减题
断言：每道题有 3 个 operands 和 2 个 operators
断言：各操作数 ≥ 2（过滤极简）
断言：中间结果 mid ∈ [0, max]
断言：最终结果 answer ∈ [0, max]
断言：按运算符顺序计算等于 answer
```

### TC-03b: 长连加连减题生成（3运算符）
```
断言：每道题有 4 个 operands 和 3 个 operators
断言：所有中间结果和最终结果 ∈ [0, max]
断言：按运算符顺序计算等于 answer
```

### TC-04: 关卡题目无重复
```
生成一关 20 道题
断言：所有题目的 operands+operators 组合唯一
```

### TC-05: 题型分布合理
```
对每种模式各生成 5 关共 100 道题
统计加减法和连加连减各自比例
断言：连加连减（含2-op和3-op）总占比 ≥ 25%
```

### TC-06: 答案分布不极端集中
```
生成一关 20 道题
统计所有答案
断言：不超过 5 道题的答案相同
```

### TC-07: 操作数不过小（过滤极简题目）
```
对每种模式各生成 500 道题
断言：加法题两个操作数均 ≥ 3
断言：减法题被减数 ≥ 5，减数 ≥ 2
断言：连加连减题各操作数 ≥ 2
```
