# 数学乐园 — 产品需求文档 (PRD)

## 1. 产品概述

### 1.1 产品名称
数学乐园

### 1.2 产品定位
面向 5-7 岁儿童的趣味数学练习 H5 小游戏，以可爱的小奶龙为主题角色，通过游戏化的答题机制让孩子在玩乐中完成每日数学练习。

### 1.3 目标用户
- 主要用户：5-7 岁学龄儿童
- 辅助用户：家长（协助启动游戏、查看成绩）

### 1.4 核心目标
- 将每日 80 道数学练习变成有趣的闯关游戏
- 通过即时反馈和鼓励机制保持孩子的学习积极性
- 适配手机和平板，随时随地可以练习

## 2. 功能需求

### 2.1 难度选择
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-01 | 进入游戏后显示难度选择界面 | P0 |
| F-02 | 提供三种模式："20以内加减法"、"50以内加减法"、"100以内加减法" | P0 |
| F-03 | 界面预留扩展位，方便后续增加"乘除法"等模式 | P1 |

### 2.2 题目生成
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-04 | 题型包含：加法、减法、进位加法、退位减法、连加连减（2个运算符）、长连加连减（3个运算符） | P0 |
| F-05 | 所有中间结果和最终答案必须在 [0, max] 范围内（max 取决于所选模式） | P0 |
| F-06 | 同一关内不出现重复题目 | P0 |
| F-07 | 题目分布：约 20% 加法、20% 减法、15% 进位/退位、27% 连加连减(2-op)、18% 长连加连减(3-op)，连加连减总占比 ≥ 45% | P0 |
| F-08 | **答案分布均匀性**：将答案范围均分为若干区间（如 20 以内分 4 区：0-5/6-10/11-15/16-20），每关 20 题中各区间的答案数量不能过度集中，单区间占比不超过 40% | P0 |
| F-09 | **题目难度下限**：过于简单的题目不出现。加法两个操作数均 ≥ 3，减法被减数 ≥ 5 且减数 ≥ 2，连加连减各操作数 ≥ 2 | P0 |
| F-09a | **排除含 0 的操作数**：所有题型的操作数均不能为 0（加/减 0 无练习价值） | P0 |
| F-09b | **答案为 0 的题目限制**：每关最多出现 1 道答案为 0 的题目，避免连续出现答案为 0 的情况 | P0 |
| F-09c | **简单题比例控制**：通过概率机制控制简单题（操作数都较小的题目）在总题量中的占比不超过约 15-20%，保留灵活性而非硬性写死下限，确保大部分题目有适当难度 | P0 |

### 2.3 答题流程
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-09 | 屏幕每次显示一道题目 | P0 |
| F-10 | 通过屏幕数字键盘（0-9）输入答案 | P0 |
| F-11 | 提供删除键和确认键（确认键作为兜底提交） | P0 |
| F-12 | 答案最多输入的位数根据模式决定（20以内2位，50以内2位，100以内3位） | P0 |
| F-12a | 当输入位数达到**当前题正确答案位数**时，自动触发判题 | P0 |
| F-13 | 答对后自动跳到下一题（间隔约 1.5 秒） | P0 |
| F-14 | 答错第一次：提示"再试一次"，倒计时重置，清空输入 | P0 |
| F-15 | 答错第二次：显示正确答案，2 秒后自动跳到下一题 | P0 |
| F-16 | 倒计时耗尽视为答错，进入答错流程 | P0 |

### 2.4 关卡系统
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-17 | 每关 20 题，共 4 关（合计 80 题） | P0 |
| F-18 | 每关结束显示结算画面：正确率、平均用时 | P0 |
| F-18a | 每关结算页提供"查看错题"按钮，与"继续下一关"/"休息一下"同级显示（非嵌套容器），点击进入独立全屏错题回顾页 | P0 |
| F-18b | **错题记录逻辑**：只要第一次答错就记录为错题（即使重试后答对），记录第一次的错误答案与正确答案 | P0 |
| F-18c | **独立错题页**：全屏展示错题列表，每条显示题目、孩子首次错误答案、正确答案；底部有"返回"按钮回到结算页 | P0 |
| F-19 | 结算画面提供"继续下一关"和"休息一下"两个选项 | P0 |
| F-20 | 选择休息时显示休息画面，点击"准备好了"继续 | P0 |
| F-21 | 4 关全部完成后显示总结算画面 | P0 |
| F-22 | 总结算画面提供"再玩一次"按钮回到开始页 | P0 |

### 2.4A 盲盒奖励系统
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-22a | 每关完成后固定触发 1 次盲盒抽取 | P0 |
| F-22b | 盲盒页出现在关卡结算页之前，抽取后再进入结算页 | P0 |
| F-22c | 盲盒奖励采用组合式随机：类型 × 主题 × 品质 | P0 |
| F-22d | 品质概率：普通 65%、稀有 28%、超稀有 7% | P0 |
| F-22e | 同一局内尽量避免连续两关抽到完全相同奖励（弱去重） | P1 |
| F-22f | MVP 仅展示当次奖励，不做跨关收藏册 | P0 |
| F-22g | **盲盒方块交互**：盲盒方块在等待点击时微微周期性抽动（吸引孩子点击）；点击方块区域或"开始抽奖"按钮均可触发抽奖 | P0 |
| F-22h | **盲盒开启动画**：触发抽奖后盲盒方块先剧烈摇晃（约 0.6-0.8s），再揭晓结果 | P0 |
| F-22i | **轮转动画丰富化**：轮转期间显示的 emoji 池扩展到 10+ 个（如 🎁✨🎊🌟💎🌈🧸🎉🌊🎵🦋🍀），每次随机打乱顺序 | P0 |
| F-22j | **抽中鼓励提示语**：抽中后直接在结果区域显示按品质分级的鼓励语（不再显示"抽中：XXX"文字，也不单独在下方显示鼓励语），侧重肯定孩子的努力和进步，每个品质准备 10+ 句随机轮换。**普通**："每天进步一点点！""数学小达人！""继续加油哦！""你做得很棒！""又完成了一关！""坚持就是胜利！""你很认真呢！""学习使你更强大！""小小数学家！""一步一个脚印！" **稀有**："数学小能手！""太厉害了！""你的努力有回报了！""越来越棒了！""思维敏捷，了不起！""你是学习的榜样！""进步神速！""聪明的小脑袋！""你让小奶龙骄傲！""运气和实力并存！" **超稀有**："你是最棒的！""不可思议的天才！""超级数学之星！""了不起了不起！""简直是数学精灵！""你拥有超能力！""传说中的学霸！""全世界最厉害的小朋友！""数学王国的勇士！""你闪闪发光！" | P0 |

### 2.5 倒计时
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-23 | 每道题限时 15 秒 | P0 |
| F-24 | 倒计时以进度条或圆环形式可视化展示 | P0 |
| F-25 | 剩余 3 秒时进度条变为红色并闪烁提醒 | P1 |

### 2.6 进度展示
| 编号 | 需求 | 优先级 |
|------|------|--------|
| F-26 | 游戏画面顶部显示当前关卡和题号（如"第1关 3/20"） | P0 |
| F-27 | 圆形进度环显示当前关卡完成进度 | P0 |

## 3. 视觉设计

### 3.1 整体风格
- 活泼可爱的卡通风格
- 柔和的黄色系/暖色系配色（呼应小奶龙黄白配色）
- 圆角设计，无尖锐元素
- 大号字体和按钮，适合儿童操作

### 3.2 小奶龙角色
- 作为游戏伴侣角色，使用 `images/` 文件夹中的奶龙图片（而非 CSS 绘制）
- 不同场景使用不同神态的图片：
  - **首页**：`nailong-sayhi.png`（打招呼，欢迎进入游戏）
  - **答对/盲盒抽奖页**：`nailong-curious.png`（好奇，看看盲盒有什么）
  - **关卡完成页**：`nailong-happy.jpg`（开心，过关庆祝）
  - **答错/错题回顾**：`nailong-unhappy.png`（不开心，鼓励再试）
  - **游戏全部完成页**：`nailong-twinkle.png`（闪亮得意，全部通关）
- 图片自适应区域大小，保持比例

### 3.3 动画效果
- 答对时：屏幕飘落彩色纸片（confetti）
- 答错时：题目区域轻微抖动
- 切换题目时：淡入上滑动画
- 关卡完成时：星星爆炸效果

### 3.4 配色方案
| 用途 | 颜色 | 色值 |

| 主背景 | 淡黄色 | #FFFDE7 |
| 次背景 | 暖橙色 | #FFF8E1 |

## 4. 音效设计

### 4.1 语音鼓励（本地音频优先，TTS 兜底）
优先播放本地 mp3 素材；若素材缺失或播放失败，再回退到浏览器 Web Speech Synthesis API（中文女声优先）。

**素材来源参考：** `GoodCoder666/py-icehappy/sound/mp3`

**环节与音效映射（MVP）**

- 按键：`click.bubble.mp3`（数字键）、`click.common.button.mp3`（功能键）
- 答对（首次）：`Eliminate1.mp3` ~ `Eliminate4.mp3` 随机
- 答对（重试后）：`Eliminate5.mp3` ~ `Eliminate8.mp3` 随机
- 答错首次：`board.mp3`
- 答错最终/超时最终：`ice_break.mp3`
- 关卡完成：`money.mp3`
- 盲盒开启：`Eliminate6.mp3` 或 `Eliminate7.mp3`
- 盲盒揭晓：普通/稀有 `money.mp3`，超稀有 `unbelievable.mp3`
- 背景音乐：答题页 `GameSceneBGM.mp3`，非答题页 `WorldSceneBGM.mp3`

**鼓励语音（可轮换）**
- `good.mp3` / `great.mp3` / `amazing.mp3` / `excellent.mp3` / `unbelievable.mp3`

对应文案（用于 TTS 回退或字幕）如下：

**答对（一次答对）：**
- "太棒了！"
- "你真厉害！"
- "答对了！好聪明！"
- "非常好！"
- "就是这样！"
- "你是最棒的！"
- "继续加油！"

**答对（第二次答对）：**
- "不放弃，真棒！"
- "坚持就是胜利！"
- "好样的！"

**答错（第一次，鼓励重试）：**
- "再试一次哦！"
- "别着急，再想想！"
- "加油，你可以的！"

**答错（两次都错）：**
- "没关系，下一题加油！"
- "没关系，下次一定行！"
- "记住答案，下次就会了！"

**倒计时耗尽：**
- "时间到啦！"
- "要快一点哦！"

**关卡完成：**
- "太厉害了！一关通过！"
- "你真棒！休息一下吧！"

### 4.2 音效 (Web Audio API)
| 场景 | 音效描述 |
|------|----------|
| 答对 | 本地 mp3 片段（Eliminate 系列） |
| 答错 | 本地 mp3 片段（board / ice_break） |
| 按键 | 本地 mp3 点击音（click.*） |
| 倒计时最后3秒 | 沿用现有轻提示音策略 |
| 关卡完成 | 本地 mp3 奖励音（money） |
| 全部完成 | 可沿用现有庆祝策略（后续可替换） |

### 4.3 背景音乐
- 使用本地 BGM：`GameSceneBGM.mp3`、`WorldSceneBGM.mp3`
- 音量极低（作为氛围底衬）
- 场景切换时淡入淡出，避免突兀切换
- 提供开关按钮
- **首页 BGM**：页面加载后立即尝试自动播放 `WorldSceneBGM.mp3`；若浏览器自动播放策略阻止，则监听用户首次点击/触摸页面任意位置时恢复播放

## 5. 交互流程图

```
开始页面（选择难度）
    │
    │ ←── 页面加载即尝试播放 WorldSceneBGM（被阻止则首次点击时播放）
    ▼
游戏页面（第1关，第1题）
    │
    ├─ 答对 → 庆祝动画 → 1.5s后下一题
    ├─ 答错(第1次) → 记录错题 → "再试一次" → 重置倒计时
    ├─ 答错(第2次) → 显示正确答案 → 2s后下一题
    └─ 倒计时耗尽 → 视为答错
    │
    ▼ (20题完成)
盲盒页面（方块微微抽动 → 点击触发 → 剧烈摇晃 → 揭晓奖励 → 直接显示鼓励语）
    │
    ▼
关卡结算页面
    │
    ├─ "查看错题"（同级按钮）→ 独立全屏错题回顾页 → "返回" → 关卡结算页
    ├─ "继续下一关" → 游戏页面（下一关）
    └─ "休息一下" → 休息页面 → "准备好了" → 游戏页面
    │
    ▼ (4关全部完成)
总结算页面
    │
    └─ "再玩一次" → 开始页面
```

## 6. 设备兼容

### 6.1 支持设备
- 手机（iOS Safari、Android Chrome）
- 平板（iPad Safari、Android Chrome）

### 6.2 响应式要求
- 手机端：竖屏全屏布局，按钮不小于 48px
- 平板端：居中显示，内容区域最大宽度 600px
- 禁止缩放（防止误触双击放大）

---

## 7. 技术约束
- 纯前端实现，单个 HTML 文件
- 无外部依赖（不依赖CDN、第三方库）
- 浏览器打开即可使用，无需安装
